# Autonomous Data Science Agent for EDA - 15-Day Plan

## Goal
بناء وكيل EDA مستقل قادر على كتابة كود Pandas، تنفيذه، قراءة النتائج، تصحيح الأخطاء ذاتياً، وتخزين ملفات تعريف البيانات لتسريع العمل.

## خطة الـ15 يوم

### الأيام 1-4: Tool Environment
1. إنشاء صندوق رمل آمن عبر Docker لتنفيذ الكود.
2. تصميم أدوات وواجهات Pydantic (Tool Inputs/Outputs).
3. إعداد هيكل backend للوكيل.
4. إنشاء dataset تجريبي وواجهة تتيح إرسال طلبات إلى الوكيل.

### الأيام 5-9: Agent Core
5. تنفيذ بنية ReAct: Observe -> Plan -> Write -> Execute -> Observe Error -> Rewrite.
6. إدخال آلية اختيار أداة ديناميكية Python / SQL.
7. تطوير كود توليد وتحليل بيانات EDA.
8. إضافة معالج أخطاء ذاتي لإعادة المحاولة.
9. اختبار الدورة الكاملة على مجموعة تجريبية.

### الأيام 10-12: Memory
10. تنفيذ ذاكرة قصيرة الأمد لحفظ نتائج `df.describe()` وخصائص البيانات.
11. تقليل الحساب المتكرر عبر الذكريات.
12. دمج الذاكرة في دورة الوكيل.

### الأيام 13-15: Evaluation
13. اختيار 5 مجموعات بيانات متنوعة.
14. قياس الأداء والمقارنة مع ChatGPT single-prompt وAutoML.
15. كتابة تقرير/كتاب يشرح كل خطوة ويعرض النتائج الرقمية.

## الحالة الحالية
- تم إعداد واجهة React بسيطة في `frontend/ai-agent`.
- تم إنشاء backend Python مع FastAPI وPydantic في `backend/`.
- تم إضافة Dockerfile وصندوق رمل للتنفيذ.
- تم تنفيذ بنية Agent Core المبدئية مع حلقة ReAct وتحسين اختيار الأدوات.
- تم إضافة دعم تقييم الأداء عبر `GET /api/evaluate` ومجموعات بيانات تجريبية متعددة.
- تحتاج البيئة لتشغيل Docker daemon لبناء وتشغيل الحاوية، أو تثبيت Python محليًا.
